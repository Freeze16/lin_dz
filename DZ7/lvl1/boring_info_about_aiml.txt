- Для языка AIML применяется файл AIML (XML) и интерпритатор, что бдет обрабатывать реплики собеседника.
- Язык AIML состоит из различных тегов, подобно всеми известным HTML или XML.

- category - содержит набор каких-то категорий (других тегов, обрабатывающих реплику)
- pattern и template - программа находит тег с нужным содержанием в "паттерне", после чего активируется "темплейт".
- * - используется для определения произвольной группы слов.
- li - ответ, в данном теге, активируемый заданным условием. Сам тег находиться в "темплейте"
- random - рандомно выбирает один из заданных тегов li
- srai - ссылка на другой "паттерн"
- star - текст, что относится к * будет передан в данных тег
- <sr/> - сокращённая запись <srai><star/></srai>
- _ - не обращает внимание на то, что было перед фразой (или после неё)
- topic - имеет в себе обязательный атрибут name, что отвечает за название темы. Нельзя использовать паттерны темы, не "войдя" в эту тему. "Войти" в эту тему можно используя тег set с обязательным атрибутом name
- action - даёт возможность боту отражать эмоции
- think - создаёт значения переменным и сохраняет их в памяти, может использоваться с тегом set.
- condition - тег условия с обязательными атрибутами name и value. Применяется с тегами li.
- learnf - автоматическое создание шаблона. Используется с тегами eval и code (позваляют выполнить то, что передано в виде строки)
- ~ - обозначает любое окончание или начало слова;
- {///} - словарь синонимов;
- <////> - наличие или отсутствие слов


- Первый виртуальным собеседником была Элиза (1966 год)
- Подобный метод использовался в программе A.L.I.C.E. (1995 год)
- AIML был создан Ричардом Уолессом.
- Существует много интерпритаторов этого языка, например Программа D
- Так же существует формальная спецификация языка и XML схема.
